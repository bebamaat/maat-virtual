### Refatoração canal WhatsApp em 3 trilhas

Em `painel/data/maat-virtual.json`, o canal `id="whatsapp"` foi substituído. Schema antigo: `etapas[]` direto (7 etapas, incluindo "Menu + Classificação"). Schema novo: `trilhas[]` com 3 trilhas, cada uma com suas próprias `etapas[]`:

- **wa-rep — Representantes** (verde `#085041`, subtítulo "Bot-Rep — alto giro, tabela fixa de transferência"). 6 etapas: Atração, Pedido (Formulário), Pagamento, Operações, Entrega, Follow-up + Recompra.
- **wa-pf — Pessoa Física** (azul `#0C447C`, subtítulo "Cliente final descobre via WA, redireciona pro Shopify"). 4 etapas: Atração, Redirect Shopify, Acompanhamento, Pós-venda.
- **wa-pj — Pessoa Jurídica** (roxo `#5B297D`, subtítulo "Cliente empresarial — alta margem, alto volume, ciclo lento"). 7 etapas: Atração, Negociação, Proposta + Pix, Pagamento, Operações, Entrega, Follow-up + Recompra.

A etapa "Menu + Classificação" do schema antigo virou pressuposto narrado na `descricao` do canal (não aparece como etapa do diagrama).

Cada etapa preserva o schema completo (`detalheNegocio`, `detalheTecnico[]`, `maatVirtualNota`).

### Renderização — 1 linha por trilha (não tabs)

Em `painel/app/page.js`, o componente `FluxoOperacionalView` agora expande canais com `trilhas[]` em N canais virtuais antes de renderizar:

```js
const canaisExpandidos = fluxoOperacional.canais.flatMap(canal => {
  if (canal.trilhas) {
    return canal.trilhas.map(trilha => ({
      id: `${canal.id}-${trilha.id}`,
      nome: `${canal.nome} · ${trilha.nome}`,
      descricao: trilha.subtitulo || canal.descricao,
      corPrincipal: trilha.corPrincipal,
      etapas: trilha.etapas,
    }));
  }
  return [canal];
});
```

Cada trilha vira sua própria linha no fluxo (igual Shopify e Mercado Livre). Border-left de 4px na `.fluxo-canal-head` na cor da trilha — apenas no head, não atravessa as etapas. Modal mostra "Canal: WhatsApp Business · Representantes" (etc).

**Tentativa anterior descartada:** primeira versão tinha tabs internas dentro do canal whatsapp (`<CanalComTrilhas>` com `useState(trilhaAtivaId)`). Visualmente os tabs pareciam cards e confundiam a leitura. Solução final é mais limpa — cada trilha é primeira-classe no diagrama.
