## Como continuar de outra maquina

### 1. Clone + setup basico
```bash
git clone https://github.com/bebamaat/maat-virtual.git
cd maat-virtual
```

### 2. Identificar o humano
Criar arquivo `.eu` na raiz com o nome em minusculo (cleber, lucas, julia, paula). Ver CLAUDE.md secao "Identificacao do humano".

### 3. Painel — dev local
```bash
cd painel
cp .env.local.example .env.local   # preencher com credenciais Google OAuth + NextAuth
npm install
npm run dev   # sync-docs roda antes
```
Variaveis necessarias em `.env.local`: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `NEXTAUTH_SECRET`, `NEXTAUTH_URL`. As de producao vivem no Vercel.

### 4. Deploy para producao
```bash
cd painel
vercel link --project painel   # primeira vez na maquina
vercel --prod
```
O dominio `virtual.bebamaat.com.br` e aliasado automaticamente.

### 5. Editar Frente A (ou outros docs)
1. Editar `docs/maat-virtual/<arquivo>.md` (fonte canonica).
2. Rodar `npm --prefix painel run sync-docs` (ou deixar o predev/prebuild rodar).
3. Commitar **os dois** arquivos: `docs/maat-virtual/<arquivo>.md` e `painel/public/docs/<arquivo>.md`.
4. Push + `vercel --prod` a partir de `painel/` para atualizar o ar.
