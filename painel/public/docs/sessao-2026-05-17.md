# Sessão 2026-05-17 — Árvore-mestra: alinhamento Mapa ↔ Organograma ↔ Time Virtual

Condutora: Julia. Claude Code Opus 4.7 (1M context). Skills usadas: `design-critique` e `design-token-audit`. Escopo: refinar a aba Mapa Estratégico — adicionar descrições de áreas/setores e, principalmente, alinhar a estrutura entre as 3 abas (Mapa = estrutura organizacional, Organograma = hierarquia, Time Virtual = JDs) para que sejam espelhos da mesma árvore.

## Diagnóstico (design-critique)

Auditoria cruzada revelou desalinhamento **maior que só "Comunidade faltando"**: o painel tinha **duas taxonomias que não conversavam** — Time Virtual com 7 coordenadores planetários soltos, Mapa Estratégico com 5 áreas próprias. "Receita" e "Produção" do Mapa eram agregações; "Comunidade" (Urano) não existia no Mapa; "Tecnologia" (criada em 14/05) não tinha coordenador.

## Decisão estrutural (aprovada pela Julia)

**Árvore-mestra única — 6 áreas em 2 camadas:**

Camada transversal:
- **Estratégia** — coordenador Júpiter (Têmis, Sofia, Fortuna)
- **Tecnologia** — coordenação Cleber/CTIO, sem planeta (nó coordenador `tecnologia` marcado `ehHumano`)

Camada de operação:
- **Receita** — Mercúrio (Vendas) + Vênus (Marketing)
- **Produção** — Marte (Produção) + Netuno (Logística)
- **ADM Financeiro** — Saturno
- **Comunidade** — Urano

Filosofia: o Mapa agrega áreas macro; os planetas viram setores dentro delas. As 3 abas derivam da mesma árvore → espelho garantido por construção.

## O que foi feito

### JSON (`maat-virtual.json`)
- Novo array **`areasMestras[]`** — 6 áreas com `id`, `nome`, `camada`, `descricao`, `coordenadores[]`. É a fonte canônica da estrutura.
- Cada coordenador ganhou campo **`areaMestra`**.
- Novo nó coordenador **`tecnologia`** (`nomeMitologico: "Cleber"`, `ehHumano: true`) — para os especialistas de Inteligência Digital poderem pendurar.
- **Maia (GA4/GTM) e Íris (Make/n8n) migraram** de Júpiter/Estratégia → Tecnologia.
- Cada especialista + governança ganhou **`ferramentasIds[]`** (IDs do catálogo de 16 ferramentas).
- `mapaEstrategico` reduzido a `{ descricaoTopo, conector }` — a estrutura agora é **derivada**, não editada à mão (era assim que "Comunidade" havia sumido).

### Render (`page.js`)
- **MapaEstrategicoView** — reescrito: deriva de `areasMestras` + `coordenadores` + `especialistas`. Cada setor é um agente real com sua descrição e ferramentas. Áreas com 2 coordenadores mostram grupos nomeados (Vendas/Marketing, Produção/Logística); Tecnologia mostra Inteligência Digital + Governança Técnica.
- **HierarquiaView** (Organograma) — passou a agrupar coordenadores por área-mestra (camada visual de área); Governança Técnica renderizada dentro da área Tecnologia.
- **TimeVirtualView** — idem: agrupa por área-mestra, com cabeçalho de área e a Governança Técnica dentro de Tecnologia.

### validate.js
- Nova validação de `areasMestras`: ids únicos, camada válida, coordenadores existentes, e todo `coordenador.areaMestra` apontando para área válida. **Build crasha se a árvore dessincronizar** — impede a regressão que originou esta sessão.

### CSS (`globals.css`)
- Classes novas: `me-area-desc`, `me-grupos`, `me-grupo`, `me-grupo-nome`, `me-setor` (agora card), `me-setor-desc`; cabeçalho de área compartilhado por Organograma e Time Virtual (`organograma-area*`, `tv-area*`). `me-areas-operacao` virou grid auto-fit (4 áreas).

## Auditoria de tokens (design-token-audit)
- Cor: 100% tokens semânticos — mantido.
- Conteúdo novo (descrições) usa `--text-secondary`/`--text-muted` + tamanhos `rem` no padrão do painel — sem token novo inventado.
