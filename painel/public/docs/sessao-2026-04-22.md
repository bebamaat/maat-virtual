- **Scroll lock gateado como pattern padrão** — forma correta pros modais que são renderizados sempre no DOM (mesmo fechados):
  ```js
  useEffect(() => {
    if (!aberto) return;
    const original = document.body.style.overflow;
    document.body.style.overflow = "hidden";
    function onKey(e) { if (e.key === "Escape") onClose(); }
    window.addEventListener("keydown", onKey);
    return () => {
      document.body.style.overflow = original;
      window.removeEventListener("keydown", onKey);
    };
  }, [aberto, onClose]);
  ```
  Aplicado em `AgenteModal` (fix original `09b3ebe`), `ToolModal` (F3), `StepModal` (F4).
- **Commits separados por intenção** — cada commit carrega um escopo (fix, feat, chore, docs). Quando uma mudança acumulou conteúdo de fases distintas no working tree, usei a dança `cp backup` → `git checkout HEAD` → aplicar só a fatia → commit → `cp` de volta → commit da fatia seguinte. Preserva histórico auditável.
- **Reconciliação F1 é fonte canônica** — `docs/maat-virtual/reconciliacao-agentes-f1.md`. Qualquer nome antigo (pipedrive, shopify, bling, ml, make, etc.) que aparecer em código ou dados deve ser resolvido por essa tabela antes de usar.

## Problemas encontrados (e como resolvemos)

- **Página sem CSS após `npm run build` com dev rodando** — F3 pareceu "quebrar" com body sem estilo. Diagnóstico: não era CSS — era `.next/` corrompido. `npm run build` sobrescreve o manifesto de chunks que o `next dev` usa; o dev server perde referência aos bundles e passa a servir 404 em `/_next/static/css/app/layout.css`, `main-app.js`, etc. HTML retorna 200, aparência de "sem CSS". **Regra operacional**: não rodar `npm run build` em `painel/` enquanto o dev está ativo. Validação de compilação pelo output do dev (compila on-demand e loga erros). `build` só antes de `vercel --prod` ou em CI. Fix quando acontecer: `kill` dev, `rm -rf painel/.next`, `npm run dev` fresh.
- **Working tree acumulado entre fases** — F2 foi validada mas não commitada antes de saltar pra F3; quando foi hora de commitar a F3, `page.js` e `globals.css` já tinham mudanças das duas fases. Solução: usar `cp backup` + `git checkout HEAD -- arquivo` + reaplicar só a fatia desejada via Edits + commit + `cp` de volta. Mesmo padrão aplicado pro scroll fix (que veio depois da F2 no working tree) e pro squash da recalibração com pendências (`git reset --soft HEAD~1` + re-commit unificado).
- **Mensagens de commit consistentes com escopo** — resistir a commitar "tudo junto" quando as mensagens propostas descrevem escopos distintos. Preserva `git log` legível.

