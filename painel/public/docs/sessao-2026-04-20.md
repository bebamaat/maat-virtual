## Problemas encontrados (e como resolvemos)

- **Git remote trocado pra SSH e depois revertido pra HTTPS** — o HTTPS ja funcionava via token do gh CLI; SSH exige chave + scope `admin:public_key` que o token do gh nao tinha. Fica como HTTPS. (Chave SSH ainda existe em `~/.ssh/id_ed25519` mas nao esta no GitHub.)
- **Dev local quebrou com `OAuthSignin`** — faltava `.env.local` na maquina nova. Resolvido com `vercel env pull` (precisa ajustar `NEXTAUTH_URL` pra `http://localhost:3000` depois).
- **Google OAuth bloqueia localhost** — `redirect_uri_mismatch`: o client OAuth no Google Cloud so tem a URL de producao (`virtual.bebamaat.com.br/api/auth/callback/google`) cadastrada. Pra destravar dev local, adicionar `http://localhost:3000/api/auth/callback/google` como redirect URI autorizada no Google Cloud Console. **Pendente.**
- **Tela de login ignorava o novo CSS** — estava com estilos inline hardcoded. Reescrita usando classes.

## Como continuar de outra maquina

### 1. Clone + setup basico
```bash
git clone https://github.com/bebamaat/maat-virtual.git
cd maat-virtual
```

### 2. Identificar o humano
Criar arquivo `.eu` na raiz com o nome em minusculo (`julia`, `cleber`, `lucas`, `paula`). Veja `CLAUDE.md` — secao "Identificacao do humano".

### 3. Painel — dev local
```bash
cd painel
npm install
vercel link --project painel           # so na primeira vez
vercel env pull .env.local --environment=production
# Ajustar NEXTAUTH_URL no .env.local pra http://localhost:3000
# Remover qualquer \n literal que o vercel env pull deixa em NEXTAUTH_SECRET / NEXTAUTH_URL
npm run dev
```

**Atencao:** o login local so vai funcionar se `http://localhost:3000/api/auth/callback/google` estiver cadastrada como redirect URI no Google Cloud Console (ver "Problemas encontrados"). Enquanto isso nao for feito, a alternativa e ver as mudancas direto em producao apos `vercel --prod`.

### 4. Deploy para producao
```bash
cd painel
vercel --prod
```

### 5. Fontes Larken e logos
Ja estao em `painel/public/fonts/larken/` e `painel/public/logos/` — commitados. Nao precisa copiar de fora do repo.
