## Lições + decisões operacionais

- **Schema `responsavel` string|array**: `pendenciasHumanas[].responsavel` e `data.tasks[].assignee` aceitam string ou array de strings. Validator `validateMaatVirtual` adaptado. Renderização decode pelo padrão `Array.isArray(x) ? x : [x]` + map por `humanosMap` + join `, `.
- **Estado UI vs estado de dados**: `data.tasks` é fonte canônica versionada via git (commits manuais). `state.json` é mutável via API do painel (commits automáticos pelo botão "marcar como respondida"). NÃO confundir. Bug recorrente: refatorar a aba pra ler de `state.X` quando deveria ler de `data.X`.
- **Cache stale do Next 14**: edits em arquivos JSON importados estaticamente às vezes não invalidam HMR. Sintoma: badge mostra um número, seção mostra outro. Fix: `lsof -ti :3000 | xargs kill -9 && rm -rf painel/.next && npm run dev`.
- **Build vs dev**: nunca rodar `npm run build` com dev vivo (compartilham `.next/`, build mata o dev e quebra CSS).
- **Doc histórico vs doc vivo**: `sessao-*.md` são imutáveis (registro de quando foi feito). Frente A/B + Constituição + reconciliacao-agentes são vivos. Renames que afetam docs históricos só com nota explicativa preservando o evento histórico (caso Afrodite→Odoyá em sessao-2026-04-22.md).
- **Deploy é manual**: auto-deploy via GitHub está DESLIGADO no projeto Vercel `painel`. Push pra main NÃO dispara deploy. Sempre `cd painel && vercel --prod`.
- **`.pages` no working tree**: arquivo do Pages (Mac) que combinamos deixar quieto — não commita, não ignora explicitamente, só ignora.
- **Conflitos no rebase**: state.json muda em produção via API. Quando push é rejeitado por commits automáticos remotos, fazer `git pull --rebase` e resolver conflito em `state.json` ficando com `--ours` (HEAD = origin) pra preservar respostas reais que humanos deram pelo painel.

## Como retomar de outra máquina

1. `git clone https://github.com/bebamaat/maat-virtual.git`
2. `cd maat-virtual`
3. Criar arquivo `.eu` na raiz com `julia` (ou outro nome) — define perfil ativo do humano. Já está no `.gitignore`.
4. Para o painel: `cd painel && cp .env.local.example .env.local` e preencher credenciais (NextAuth + GitHub token), `npm install`, `npm run dev`.
5. Para deploy: `cd painel && vercel link --project painel` (só primeira vez), depois `vercel --prod`.
6. Materiais visuais (`materiais/`) ficam fora do git — pedir cópia local pelo Drive da MAAT.
